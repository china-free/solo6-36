
## 1. 产品概述
本系统是一个企业内部请假申请管理系统，旨在简化员工请假流程，提高审批效率，方便员工和管理者进行请假申请与审批操作。
- 主要功能：员工提交请假申请、管理者审批申请、年假统计
- 目标用户：企业员工和部门管理者

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 员工 | 系统预置 | 提交请假申请、查看自己的申请记录及状态、查看年假统计 |
| 管理者 | 系统预置 | 查看所有待审批申请、通过/拒绝请假申请、查看所有员工年假统计 |

### 2.2 功能模块
1. **角色切换模块**：下拉选择角色（员工/管理者），切换不同的操作界面
2. **员工请假申请模块**：填写请假日期、类型、天数、理由，提交申请
3. **员工申请记录模块**：查看自己提交的所有申请及审批状态
4. **管理者审批模块**：查看待审批申请列表，进行通过/拒绝操作
5. **年假统计模块**：统计各员工已休年假天数和剩余年假天数

### 2.3 页面详情
| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|------------|
| 主页面 | 角色切换栏 | 顶部下拉选择框，切换员工/管理者角色 |
| 员工视图 | 请假申请表单 | 包含日期选择、类型选择（年假/事假/病假）、天数输入、理由输入、提交按钮 |
| 员工视图 | 我的申请列表 | 显示员工提交的所有申请，显示状态标签、详情信息 |
| 管理者视图 | 待审批列表 | 显示所有待审批的请假申请，包含申请人、日期、类型、天数、理由 |
| 管理者视图 | 审批操作 | 通过按钮、拒绝按钮 |
| 统计视图 | 年假统计表 | 员工姓名、已休天数、剩余天数、总年假天数 |

## 3. 核心流程

员工提交请假申请后，申请自动进入"待审批"状态。管理者查看待审批列表，点击"通过"或"拒绝"更新申请状态。员工可随时查看自己申请的最新状态。系统自动统计已休年假。

```mermaid
flowchart LR
    A["员工填写请假申请表单 --> B["提交申请"]
    B --> C["申请进入待审批列表"]
    C --> D["管理者查看待审批列表"]
    D --> E{"审批决定"}
    E -->|通过| F["状态更新为已通过"]
    E -->|拒绝| G["状态更新为已拒绝"]
    F --> H["员工查看申请状态"]
    G --> H
    H --> I["系统更新年假统计"]
```

## 4. 用户界面设计

### 4.1 设计风格
- 主色调：深蓝色（#1e40af）配浅灰背景，体现专业企业氛围
- 按钮风格：圆角按钮，悬停有阴影和颜色渐变效果
- 字体：使用现代无衬线字体，层次分明
- 布局风格：卡片式布局，顶部导航栏，内容区使用网格布局
- 图标风格：使用 Lucide 图标库，简洁现代

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 主页面 | 顶部导航 | 深蓝色背景，白色文字，角色选择下拉框 |
| 员工视图 | 申请表单 | 白色卡片，表单标签左对齐，输入框带边框聚焦效果 |
| 员工视图 | 申请列表 | 卡片列表，状态标签用不同颜色区分 |
| 管理者视图 | 审批列表 | 表格布局，操作按钮组 |
| 统计视图 | 统计表格 | 带进度条的统计展示 |

### 4.3 响应性
- 桌面端优先设计，移动端自适应布局调整，触摸区域优化
